# Project Context
Last updated: 2026-05-30
Version: 1.0

## Project
Name: CCCUPXLI - Homepage Frontend
What it does: The main landing page for cccup.id, focusing on high conversion, aesthetics, and user onboarding.
Stage: MVP / In Development
Repo: local/cc-cup-frontend/cc-cup-xli-homepage

## Stack
Runtime: Node.js
Framework: React 19 (via Vite)
Language: TypeScript
Styling: Tailwind CSS v4

## Structure
src/
  App.tsx         <- Main landing page layout
  index.css       <- Global styles & Tailwind imports
  components/     <- (TBD) Reusable UI components
context/
  30-05-26.md     <- Context & decision logs (This file)

## Environment & Hosting
Deployment: Static CDN (Vercel/Netlify recommended)
API Connection: Django Backend (cc_cup_xli)

## Commands
Install: npm install
Dev: npm run dev
Build: npm run build

## Conventions
- Use functional React components with hooks.
- Enforce strict typing with TypeScript.
- Use Tailwind for all styling; avoid custom CSS unless absolutely necessary for complex animations.

## Decisions Log
- **2026-05-30: Framework Choice:** Chose React + Vite over Next.js because the landing page is primarily static/client-side and doesn't currently require complex SSR.
- **2026-05-30: Styling:** Opted for Tailwind CSS v4 to enforce a cohesive, modern, and dynamic design system quickly.

## Gotchas
- Tailwind v4 uses `@import "tailwindcss";` in CSS rather than `tailwind.config.js`.

## Don't
- Don't introduce state management libraries (like Redux/Zustand) until prop-drilling becomes an actual issue.
- Don't mix JSX and TSX in this module; keep everything TypeScript.
