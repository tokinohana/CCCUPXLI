# Project Context
Last updated: 2026-05-30
Version: 1.0

## Project
Name: CCCUPXLI - User Module
What it does: User management, custom user models, and authentication handling for the CCCUPXLI system.
Stage: MVP / In Development
Repo: local/cc_cup_xli

## Stack
Runtime: Python 3
Framework: Django 6.0
API Framework: Django REST Framework (DRF)

## Structure
user/
  models.py       <- Custom user models (AUTH_USER_MODEL)
  views.py        <- User-related views and logic
  handling.md     <- Documentation on user handling
  models.md       <- Documentation on user models

## Database
DB: PostgreSQL (via dj-database-url)
ORM: Django ORM
Schema location: user/models.py
Migrations: user/migrations/
Run migrations: python manage.py migrate

## Auth
Provider: Django Authentication / SimpleJWT
Session access: request.user
User object: user.User
Protected routes: DRF Permissions / Django Auth

## Env Vars
DATABASE_URL
SECRET_KEY
DEBUG

## Commands
Install: pip install -r requirements.txt
Dev: python manage.py runserver
Migrations: python manage.py makemigrations user
Apply: python manage.py migrate

## Conventions
- Use Django ORM for all database operations.
- Follow DRF standards for API endpoints.
- Keep business logic decoupled where possible.

## Services
Database: PostgreSQL

## Gotchas
- AUTH_USER_MODEL is set to 'user.User', do not use the default Django User model.

## Don't
- Don't bypass the Django ORM for database queries.
- Don't introduce async frameworks unless explicitly requested (Django is primarily synchronous here).
- Don't rewrite working authentication code without permission.
