# Project Context
Last updated: 2026-05-30
Version: 1.0

## Project
Name: CCCUPXLI - Ticketing Module
What it does: Handles ticketing, datasets, and ticket-related logic for the CCCUPXLI system.
Stage: MVP / In Development
Repo: local/cc_cup_xli

## Stack
Runtime: Python 3
Framework: Django 6.0
API Framework: Django REST Framework (DRF)

## Structure
ticketing/
  models.py       <- Ticketing models
  views.py        <- Ticketing views/APIs
  datasets/       <- Dataset storage for ticketing
  secrets/        <- Secret handling for tickets
  plan.md         <- Ticketing specific planning docs

## Database
DB: PostgreSQL (via dj-database-url)
ORM: Django ORM
Schema location: ticketing/models.py
Migrations: ticketing/migrations/
Run migrations: python manage.py migrate

## Auth
Provider: Django Authentication / SimpleJWT
Session access: request.user

## Env Vars
DATABASE_URL
SECRET_KEY
DEBUG

## Commands
Install: pip install -r requirements.txt
Dev: python manage.py runserver
Migrations: python manage.py makemigrations ticketing

## Conventions
- Use Django ORM for all database operations.
- API endpoints should use Django Rest Framework.

## Services
Database: PostgreSQL

## Gotchas
- Ensure ticket datasets and secrets are handled securely.

## Don't
- Don't leak ticket secrets in logs.
- Don't bypass the Django ORM.
