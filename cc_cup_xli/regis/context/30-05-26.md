# Project Context
Last updated: 2026-05-30
Version: 1.0

## Project
Name: CCCUPXLI - Regis Module
What it does: Handles registrations, chatbot layout/moderation/routes, and registration forms.
Stage: MVP / In Development
Repo: local/cc_cup_xli

## Stack
Runtime: Python 3
Framework: Django 6.0
API Framework: Django REST Framework (DRF)

## Structure
regis/
  models.py             <- Registration models
  views.py              <- Registration logic and views
  REGIS_FORMS.md        <- Form definitions/docs
  REGIS_ROUTES.md       <- Route definitions/docs
  CHATBOT_layout.md     <- Chatbot layout docs
  CHATBOT_moderation.md <- Chatbot moderation docs
  CHATBOT_route.md      <- Chatbot routing docs

## Database
DB: PostgreSQL (via dj-database-url)
ORM: Django ORM
Schema location: regis/models.py
Migrations: regis/migrations/
Run migrations: python manage.py migrate

## Auth
Provider: Django Authentication / SessionAuthentication (for Regis/Admin)
Session access: request.user

## Env Vars
DATABASE_URL
SECRET_KEY
DEBUG

## Commands
Install: pip install -r requirements.txt
Dev: python manage.py runserver
Migrations: python manage.py makemigrations regis

## Conventions
- Rely on DRF for robust API creation.
- Session Authentication is primarily used for the Regis module.

## Services
Database: PostgreSQL

## Gotchas
- This module relies on SessionAuthentication as configured in REST_FRAMEWORK settings.

## Don't
- Don't break SessionAuthentication endpoints by forcing JWT where not needed.
- Don't bypass the Django ORM.
