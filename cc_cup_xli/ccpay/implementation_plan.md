# CC Pay Backend Payment & Merchant Dashboard Plan

This plan addresses:
1. Implementing payment idempotency on `ProcessPaymentView` to prevent double-charging.
2. Linking transactions to `MerchantStand` records.
3. Creating the Merchant Dashboard API endpoints to calculate accumulated balances, daily transaction history, and live transactions.

## User Review Required

> [!IMPORTANT]
> To implement idempotency, the merchant scanner app (frontend) must generate a unique ID (UUID) for each payment session before requesting deduction. We will call this the `reference_id`.
> If the merchant's device retries the network request with the same `reference_id`, the backend will check for it and return the successful status of the previous attempt without deducting the student's balance twice.

## Proposed Changes

### [Component: Database Model Updates]

#### [MODIFY] [models.py](file:///home/benhal/Documents/webdev/CCCUPXLI/cc_cup_xli/ccpay/models.py)
Add `merchant_stand` relationship and `reference_id` to the `Transaction` model:
```python
class Transaction(models.Model):
    # Existing fields...
    merchant_stand = models.ForeignKey('MerchantStand', on_delete=models.SET_NULL, null=True, blank=True, related_name='transactions')
    reference_id = models.UUIDField(null=True, blank=True, unique=True, help_text="Unique reference generated by the frontend for idempotency")
```

### [Component: API Serialization & Logic Updates]

#### [MODIFY] [serializers.py](file:///home/benhal/Documents/webdev/CCCUPXLI/cc_cup_xli/ccpay/serializers.py)
- Update `PaymentRequestSerializer` to require `reference_id` (a UUID).
- Update `TransactionSerializer` to include `merchant_stand` and `reference_id`.

#### [MODIFY] [views.py](file:///home/benhal/Documents/webdev/CCCUPXLI/cc_cup_xli/ccpay/views.py)
- **`ProcessPaymentView`**:
  - Before running the debit transaction, query for an existing transaction with the given `reference_id`.
  - If it exists, return the existing transaction response immediately with HTTP 200 (Success).
  - Otherwise, run the atomic debit and save `merchant_stand=merchant` and `reference_id=reference_id` to the transaction ledger.
- **New `MerchantDashboardView`** (Endpoint: `api/ccpay/merchant/dashboard/`):
  - Requires a merchant `token` query parameter.
  - Returns:
    - Current merchant name and status.
    - Total accumulated revenue today.
    - Daily summary: Accumulated revenue split by days (summing transactions grouped by date).
    - Live feed: Today's transactions list (ordered by time descending).
